# GitHub Repository Monitor

Продвинутый мониторинг GitHub репозиториев с умной системой уведомлений в Telegram. Вдохновлено лучшими практиками из 4ifir-checker.

## 🚀 Возможности

### Основные функции
- **Умный мониторинг**: Проверяет наличие новых изменений перед запуском
- **Экономия ресурсов**: Не запускает полный мониторинг если нет изменений
- **Автоматические уведомления**: Отправка отчетов каждые 30 минут
- **Сохранение состояния**: Запоминает дату последней проверки

### Информация о репозиториях
- **Коммиты**: Последние 5 коммитов с красивым форматированием
- **Pull Requests**: Открытые PRs с детальной информацией
- **Релизы**: Информация о последних релизах
- **Workflow runs**: Статус CI/CD процессов
- **Измененные файлы**: Дерево файлов со статистикой изменений
- **Статистика**: Общая информация о звездах, форках и активности

### Форматирование
- **HTML разметка**: Профессиональное форматирование как в 4ifir-checker
- **Кодовые блоки**: `<pre><code>` для лучшей читаемости
- **Деревья файлов**: Визуальная структура изменений
- **Разделители**: Красивые `<hr>` между секциями
- **Полностью на русском**: Все сообщения на русском языке

### Telegram интеграция
- **Поддержка тем**: `TELEGRAM_TOPIC_ID` для отправки в конкретные темы
- **Fallback механизms**: Надежная доставка сообщений
- **Разбиение длинных сообщений**: Автоматическое разделение при необходимости

## 📋 Установка и настройка

### 1. Создание Telegram бота

1. Найдите в Telegram бота [@BotFather](https://t.me/botfather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания нового бота
4. Сохраните **токен бота** (выглядит как `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

### 2. Получение вашего Chat ID

**Способ 1: Через @userinfobot**
1. Найдите в Telegram бота [@userinfobot](https://t.me/userinfobot)
2. Отправьте ему любое сообщение
3. Бот пришлет ваш **Chat ID** (число)

**Способ 2: Через @RawDataBot**
1. Найдите в Telegram бота [@RawDataBot](https://t.me/RawDataBot)
2. Отправьте ему любое сообщение
3. В ответе найдите поле `"chat": {"id": 123456789}` - это ваш Chat ID

**Способ 3: Через API Telegram**
1. Найдите вашего бота в Telegram и отправьте ему сообщение
2. Перейдите по ссылке: `https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates`
3. В ответе найдите `"chat": {"id": 123456789}`

### 3. Создание GitHub Personal Access Token

1. Перейдите в [GitHub Settings > Developer settings > Personal access tokens](https://github.com/settings/tokens)
2. Нажмите "Generate new token"
3. Дайте имя токену (например, "Repository Monitor")
4. Выберите срок действия
5. Установите галочки:
   - ✅ `repo` (полный доступ к репозиториям)
   - ✅ `read:org` (если у вас есть организации)
6. Нажмите "Generate token"
7. **Скопируйте и сохраните токен** (он больше не будет показан)

### 4. Настройка репозитория

1. Создайте новый репозиторий на GitHub с названием `Ryazha-ckeker`
2. Склонируйте репозиторий или загрузите в него файлы из этого проекта
3. Перейдите в настройки репозитория: `Settings` → `Secrets and variables` → `Actions`

### 5. Добавление секретов в GitHub

В разделе "Repository secrets" добавьте следующие секреты:

| Имя секрета | Значение |
|-------------|----------|
| `G_TOKEN` | Ваш Personal Access Token |
| `TELEGRAM_BOT_TOKEN` | Токен вашего Telegram бота |
| `TELEGRAM_CHAT_ID` | Ваш Chat ID в Telegram (число) |
| `G_USERNAME` | `Dimasick-git` (ваш GitHub username) |
| `TELEGRAM_TOPIC_ID` | ID темы в Telegram (опционально) |

## ⚙️ Настройка

### Изменение интервала проверки

Откройте файл `.github/workflows/monitor.yml` и измените расписание:

```yaml
schedule:
  # Каждые 30 минут
  - cron: '*/30 * * * *'
  
  # Каждый час
  # - cron: '0 * * * *'
  
  # Каждые 2 часа
  # - cron: '0 */2 * * *'
```

### Настройка Telegram тем

Если вы хотите отправлять сообщения в конкретную тему Telegram:

1. Создайте тему в вашем Telegram чате/группе
2. Получите ID темы (обычно начинается с цифр)
3. Добавьте `TELEGRAM_TOPIC_ID` в секреты GitHub

### Кастомизация сообщения

Отредактируйте класс `MessageBuilder` в файле `main.py` для изменения формата сообщений.

### Изменение лимитов

В начале файла `main.py` можно изменить константы:

```python
MAX_ACTIVE_REPOS   = 10  # Показывать репозиториев
MAX_COMMITS        = 5   # Коммитов на репозиторий
MAX_PRS            = 3   # Pull Requests
MAX_RELEASES       = 1   # Релизов
MAX_WORKFLOWS      = 3   # Workflow runs
```

## 🧪 Тестирование

### Тестирование Telegram бота

Специальный workflow для тестирования только Telegram:

1. Перейдите в `Actions` → `Test Telegram`
2. Нажмите `Run workflow`
3. Проверьте получение тестового сообщения

Для тестирования можно запустить скрипт локально:

```bash

python main.py
```

## 🚨 Важные замечания

### Лимиты GitHub Actions
- Бесплатные аккаунты: 2000 минут в месяц
- Мониторинг каждые 30 минут: ~1440 минут в месяц
- Умная проверка экономит ресурсы при отсутствии изменений

### Безопасность
- Храните все токены и секреты в GitHub Secrets
- Используйте Personal Access Token с минимальными правами
- Регулярно обновляйте токены безопасности

### Производительность
- Система автоматически пропускает проверку если нет изменений
- Сохраняет состояние в `last_check.json`
- Использует rate limiting для GitHub API

## 🤝 Поддержка

Если возникли проблемы:

1. **Проверьте секреты**: Убедитесь что все переменные окружения правильные
2. **Протестируйте Telegram**: Запустите `Test Telegram` workflow
3. **Проверьте логи**: Посмотрите детальные логи в GitHub Actions
4. **Проверьте права**: Убедитесь что G_TOKEN имеет доступ к репозиториям

- **Не приходят сообщения**: Проверьте Chat ID и токен бота
- **Ошибка API**: Убедитесь что токен имеет права `repo`
- **Rate limit**: Система автоматически обрабатывает лимиты GitHub API
